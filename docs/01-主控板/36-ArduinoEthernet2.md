### **概述**

Arduino Ethernet Shield 2（以太网扩展板 2）可在几分钟内将您的 Arduino 连接到互联网。只需将这个模块插到您的 Arduino 板上，用一根 RJ45 网线（不随附）将其连接到您的网络，再遵循几个简单的步骤，即可开始通过互联网控制您的世界。与所有 Arduino 产品一样，该平台的所有元素——硬件、软件和文档——都是免费提供并开源的。这意味着您可以确切了解它的制造原理，并将其设计作为您自己电路的起点。全球已有成千上万的 Arduino 板每天都在激发人们的创造力。立即加入我们，Arduino 因您而精彩！
*需要一块 Arduino 主板（不随附）

*   **工作电压：** 5V（由 Arduino 主板提供）
*   **以太网控制器：** 带有内部 32K 缓冲器的 W5500
*   **连接速度：** 10/100Mb
*   **与 Arduino 连接方式：** 通过 SPI 端口

### **技术规格**

Arduino Ethernet Shield 2 允许 Arduino 主板连接到互联网。它基于（Wiznet W5500 以太网芯片）。Wiznet W5500 提供了一个能够支持 TCP 和 UDP 的网络 (IP) 协议栈。它支持多达八个同时的套接字连接。使用 Ethernet 库来编写通过此扩展板连接到互联网的草图（Sketch）。

Ethernet Shield 2 使用穿过扩展板的长绕排针与 Arduino 主板连接。这保持了引脚布局的完整，并允许将另一个扩展板堆叠在其之上。

该板的最新修订版在 Arduino UNO 板 rev 3 版本上采用了 1.0 引脚布局。

Ethernet Shield 2 有一个标准的 RJ-45 接口，集成了线路变压器并支持**以太网供电 (Power over Ethernet - PoE)**。

板上有一个 **micro-SD 卡槽**，可用于存储文件并通过网络进行服务。它与 Arduino Uno 和 Mega（使用 Ethernet 库）兼容。板上的 micro-SD 卡读卡器可通过 **SD 库**进行访问。使用此库时，片选 (SS) 引脚为**引脚 4**。该扩展板的原始版本包含一个全尺寸 SD 卡插槽；当前版本不支持全尺寸SD卡。

该扩展板还包括一个**复位控制器**，以确保 W5500 以太网模块在通电时能正确复位。早期版本的扩展板与 Mega 不兼容，需要在通电后手动复位。

当前的扩展板支持一个旨在从传统的 5 类双绞线以太网电缆中获取电力的**以太网供电 (PoE)** 模块。
PoE 模块特性如下：

*  符合 IEEE802.3af 标准
*  输入电压范围 36V 至 57V
*  过载和短路保护
*  12V 输出
*  高效 DC/DC 转换器：典型效率 85% @ 80% 负载
*  1500V 隔离（输入到输出）

**注意：** 以太网供电模块并非由 Arduino 制造的专有硬件，它是第三方附件。更多信息，请参阅数据手册。

该扩展板**不随附内置的 PoE 模块**，它是一个必须额外添加的独立组件。

Arduino 通过 **SPI 总线**（通过 ICSP 接头）与 W5500 和 SD 卡进行通信。在 Uno 上，SPI 使用数字引脚 **10、11、12 和 13**；在 Mega 上，使用引脚 **50、51 和 52**。在这两种主板上，**引脚 10** 用于选择 W5500，**引脚 4** 用于选择 SD 卡。**这些引脚不能用于通用 I/O**。在 Mega 上，硬件 SS 引脚 **53** 不用于选择 W5500 或 SD 卡，但**必须将其保持为输出状态**，否则 SPI 接口将无法工作。

请注意，由于 W5500 和 SD 卡共享 SPI 总线，**一次只能有一个设备处于活动状态**。如果您在程序中同时使用两个外设，相应的库会处理这个问题。但是，如果您在程序中未使用其中一个外设，则需要显式地取消选择它。对于 SD 卡，请将引脚 4 设置为输出并向其写入高电平。对于 W5500，将数字引脚 10 设置为高电平输出。

该扩展板提供一个标准的 RJ45 以太网插孔。
扩展板上的复位按钮会同时复位 W5500 和 Arduino 主板。
扩展板包含多个状态指示灯 LED：

*   **ON（亮起）：** 表示主板和扩展板已通电
*   **13（引脚13）：** Arduino 标准内置 LED
*   **ACT（黄色）：** 当存在 RX 或 TX 活动时闪烁
*   **LINK（绿色）：** 表示存在网络连接，并在扩展板发送或接收数据时闪烁
